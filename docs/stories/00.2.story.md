# Story 00.2: Integração Kiwify como canal de faturamento

**Status:** Ready
**Epic:** Legal & Integrations (Etapa 00 — Utilidades)
**Traces to:** Extensão da Etapa 8 — Integração WooCommerce / Canal de Faturamento

## User Story

Como gestora ou diretora, quero conectar a Kiwify como um canal de faturamento no AdInsight para que as vendas realizadas na plataforma apareçam no dashboard de receita junto com as lojas WooCommerce.

## Acceptance Criteria

- [ ] Novo `source_type = 'kiwify'` suportado na tabela `woo_stores` (migration)
- [ ] Novos campos de credencial para Kiwify: `kiwify_client_id_encrypted`, `kiwify_client_secret_encrypted`, `kiwify_account_id_encrypted` na tabela `woo_stores` (migration)
- [ ] `kiwify.adapter.ts` implementado com autenticação OAuth2 (`client_id` + `client_secret`) e sincronização de pedidos via `account_id`
- [ ] Endpoint `POST /api/woo-stores/:id/test-connection` funciona para lojas Kiwify
- [ ] Endpoint `POST /api/woo-stores/:id/sync` sincroniza pedidos Kiwify para `woo_orders`
- [ ] `WooSourceType` atualizado para `'woocommerce' | 'manual' | 'kiwify'` em `woo-stores.types.ts` e `shared-types`
- [ ] Tela Admin → Lojas exibe opção "Kiwify" no seletor de tipo de integração ao criar nova loja
- [ ] Card Kiwify mostra os 3 campos de credencial: **Client ID**, **Client Secret**, **Account ID**
- [ ] Card Kiwify exibe status de conexão, última sincronização e botões "Testar" / "Sincronizar"
- [ ] Pedidos Kiwify aparecem no `/revenue` (tabela de pedidos e KPI cards) com o filtro de loja aplicável
- [ ] `woo-stores.service.ts` roteia sync e test-connection para `kiwify.adapter.ts` quando `source_type = 'kiwify'`
- [ ] Credenciais Kiwify criptografadas com AES-256 antes de salvar no banco (padrão existente)

## Technical Notes

- **Autenticação Kiwify:** OAuth2 client credentials flow
  ```
  POST https://api.kiwify.com.br/oauth/token
  { client_id, client_secret, grant_type: 'client_credentials' }
  → { access_token, expires_in }
  ```
- **Pedidos:** `GET https://api.kiwify.com.br/v1/orders?account_id={account_id}&page=1&limit=100`
- **Mapeamento de status Kiwify → WooOrderStatus:**
  - `paid` → `completed`
  - `waiting_payment` → `pending`
  - `refunded` → `refunded`
  - `chargedback` → `refunded`
  - `refused` → `failed`
- **Credenciais no banco:** usar prefixo `kiwify_` para distinguir dos campos WooCommerce existentes. Os campos `consumer_key_encrypted` e `consumer_secret_encrypted` permanecem para WooCommerce — não reutilizar.
- **Migration:** `20260422_000019_add_kiwify_credentials.sql`
- **Roteamento no service:** verificar `store.source_type` antes de chamar adapter:
  ```typescript
  if (store.source_type === 'kiwify') return kiwifyAdapter.sync(...)
  if (store.source_type === 'woocommerce') return wooAdapter.sync(...)
  ```
- **Frontend:** seguir exatamente o padrão do card WooCommerce em `WooStoresPage.tsx` — mesma estrutura visual, campos diferentes
