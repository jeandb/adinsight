# Story 00.1: Privacy Policy Page

**Status:** InReview
**Epic:** Legal & Compliance (Etapa 00 — Utilidades)
**Traces to:** LGPD compliance requirement — `docs/privacy-policy-adinsight.md`

## User Story

As any visitor (authenticated or not), I want to access AdInsight's privacy policy at a public URL so that I can read how my personal data is handled in accordance with the LGPD.

## Acceptance Criteria

- [x] Route `/privacy-policy` is public — accessible without authentication (no `RequireAuth` wrapper)
- [x] Page renders all 14 sections from `docs/privacy-policy-adinsight.md` with correct hierarchy (h1, h2, lists)
- [x] "Última atualização" date (2026-04-20) is visible near the top of the page
- [x] Page is readable on desktop and mobile (responsive layout, max-width container, adequate line spacing)
- [x] Login page (`/login`) includes a footer link "Política de Privacidade" pointing to `/privacy-policy`
- [x] Navigating to `/privacy-policy` while authenticated does NOT redirect to `/dashboard` (public access always allowed)
- [x] `<title>` of the page reads "Política de Privacidade — AdInsight"

## Scope

**IN:**
- New feature folder `apps/web/src/features/legal/`
- Component `PrivacyPolicyPage.tsx` with static content derived from `docs/privacy-policy-adinsight.md`
- Public route `/privacy-policy` added to `apps/web/src/router.tsx` (outside `RequireAuth`)
- Footer link on `LoginPage.tsx`
- Optional: "← Voltar para o login" back-link when user is not authenticated

**OUT:**
- Cookie consent banner (separate story if needed)
- Terms of Service page
- Backend endpoint — content is static, no API required
- Internationalization

## Dependencies

- None — this story is standalone

## Technical Notes

- Content should be hardcoded as JSX inside `PrivacyPolicyPage.tsx` to avoid adding a markdown parser dependency. Convert the markdown headings and lists to their JSX equivalents using standard Tailwind prose-friendly classes.
- The `BootstrapCheck` wrapper in the router redirects to `/setup` if `needsSetup` is true — the privacy policy route must be placed **before** `BootstrapCheck` to remain accessible in all states. Alternatively, add it inside `BootstrapCheck` but ensure `RequireAuth` is never applied.
- Use `prose` Tailwind class (if `@tailwindcss/typography` is available) or equivalent manual spacing for readability. If the plugin is absent, use `space-y-4`, `text-sm`, `leading-relaxed` manually.
- File naming follows the project convention: `PrivacyPolicyPage.tsx` (PascalCase, named export)
