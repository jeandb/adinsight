# Story 02.2: Cards de resumo de pedidos por status na página de Faturamento

**Status:** InReview
**Epic:** Revenue Dashboard
**Traces to:** PRD Seção 8 — Integração WooCommerce / Dashboard de Faturamento

## User Story

Como gestora de tráfego ou diretora, quero ver um resumo visual dos pedidos agrupados por status (completos, processando, cancelados, reembolsados) antes da tabela de pedidos, para entender em segundos quanto foi faturado, quanto está pendente e quanto se perdeu no período.

## Acceptance Criteria

- [x] 4 cards aparecem acima da tabela de pedidos: **Completos**, **Processando**, **Cancelados**, **Reembolsados**
- [x] Cada card exibe: contagem de pedidos + valor total em R$ (exceto "Processando" que exibe só contagem)
- [x] "Cancelados" exibe também o percentual em relação ao total de pedidos do período
- [x] Os cards respeitam o filtro de período ativo na página (`after`/`before`)
- [x] Os dados vêm de um endpoint dedicado `GET /revenue/orders-summary` (não da tabela paginada)
- [x] Loading state: 4 skeletons enquanto carrega
- [x] Se não houver pedidos no período, os cards exibem zero sem quebrar

## Technical Notes

- Novo endpoint backend: `GET /revenue/orders-summary?after=&before=`
- Query SQL: `SELECT status, COUNT(*) AS count, SUM(total_cents) AS total_cents FROM woo_orders WHERE order_date >= $1 AND order_date <= $2 GROUP BY status`
- Frontend: novo `useQuery` com `queryKey: ['revenue', 'orders-summary', period, dateFrom, dateTo]`
- Respeitar os mesmos `after`/`before` já calculados por `resolveRevenuePeriod` no `RevenuePage`
- Não alterar a tabela paginada existente
