# Story 02.1: Filtro de período na página de Faturamento igual ao Dashboard

**Status:** Ready
**Epic:** Revenue Dashboard
**Traces to:** PRD Seção 8 — Integração WooCommerce / Dashboard de Faturamento

## User Story

Como gestora de tráfego ou diretora, quero filtrar o faturamento pelo mesmo conjunto de períodos disponíveis no dashboard (incluindo período personalizado), e que o filtro persista na URL, para que eu possa comparar dados de períodos específicos e compartilhar a visualização com a equipe.

## Acceptance Criteria

- [ ] A página `/revenue` exibe os mesmos 6 períodos do dashboard: "Últimos 7 dias", "Últimos 14 dias", "Últimos 30 dias", "Este mês", "Mês anterior", "Período personalizado"
- [ ] Ao selecionar "Período personalizado", dois campos de data (`date_from` / `date_to`) aparecem para seleção
- [ ] O período selecionado persiste na URL como query param (`?period=last_30d` ou `?period=custom&date_from=...&date_to=...`)
- [ ] Ao recarregar a página ou copiar a URL, o filtro selecionado é restaurado corretamente
- [ ] Todos os dados da página (KPIs, gráficos, tabela de pedidos) são atualizados ao trocar o período
- [ ] O período padrão ao acessar `/revenue` sem params é `last_30d`
- [ ] Os 3 botões antigos (7d / 30d / 90d) são removidos

## Technical Notes

- Backend já aceita `after` e `before` como ISO strings — nenhuma alteração necessária no backend
- Reutilizar `PERIOD_LABELS` e a lógica de resolução de período já existente no dashboard
- Usar `useSearchParams` (react-router-dom) para persistência na URL — mesma abordagem de `use-filters.ts`
- Período `last_90d` do antigo filtro não existe no dashboard; substituir por `this_month` e `last_month`
